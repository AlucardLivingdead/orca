Filing Bugs

The following sections provide information on where to file bugs and any additional information, such as keywords and CC lists that you should add to the bug report. If you do file a bug, please make sure you update the table above.

Firefox:

  • URL: https://bugzilla.mozilla.org/enter_bug.cgi?product=Firefox
  • CC: browser-china-atf@sun.com
  • Keywords: Always add "access", and add "sec508" if it relates to any of the 12 items in Section 508 or if it prevents the application from being used at all.

Evolution:

  • URL: http://bugzilla.gnome.org/enter_bug.cgi
  • CC: sceri-evolution-acc@Sun.COM, gnome-access-bugs@basso.sfbay.sun.com
  • Keywords: "accessibility"

GNOME/GTK+-2.0:

  • URL: http://bugzilla.gnome.org/enter_bug.cgi
  • CC: gnome-access-bugs@basso.sfbay.sun.com
  • Keywords: "accessibility"
  • NOTE: ATK is the application side of things, GAIL provides the ATK implementation for GTK, AT-SPI is the assistive technology side of things and also provides the atk-bridge and AT-SPI registry. When entering a bug, give some thought into where the bug might lie.

OpenOffice:

  • URL: http://bugzilla.gnome.org/enter_bug.cgi
  • CC: TBD.
  • Keywords: "accessibility"
